# 慧阅系统——前后端功能需求文档

## 项目概述

**慧阅——图书馆 AI 低碳智控系统**，基于 JACE8000 + IO-22U + 工业传感器。所有传感器数据经 IO-22U → JACE8000 → oBIX 协议暴露，PC 端后端读取 oBIX 数据并提供 API，前端 Dashboard 展示和控制。

**架构**：后端中转模式（后端读 oBIX → 提供 RESTful API → 前端调 API），PC 端 AI 双引擎决策，本地部署浏览器访问。

---

## 一、硬件资源清单

### 传感器（8 路数据）

| # | 传感器 | 输出信号 | IO-22U 通道 | oBIX 点名 |
|---|--------|---------|------------|-----------|
| 1 | 温湿度变送器（温度） | 4-20mA | UI-1 | `temp_sensor` |
| 2 | 温湿度变送器（湿度） | 4-20mA | UI-2 | `humidity_sensor` |
| 3 | 光照传感器 | 4-20mA | UI-3 | `light_sensor` |
| 4 | CO₂ 传感器 | 4-20mA | UI-4 | `co2_sensor` |
| 5 | 噪声传感器 | 4-20mA | UI-5 | `noise_sensor` |
| 6 | 烟雾传感器 | 4-20mA | UI-6 | `smoke_sensor` |
| 7 | PM2.5 传感器 | 4-20mA | UI-7 | `pm25_sensor` |
| 8 | 电流变送器（风扇回路） | 4-20mA | UI-8 | `fan_current` |

> 电流变送器串联在风扇供电回路中，功率 = 24V × 电流值。

### 输出设备（4 个）

| # | 设备 | 控制方式 | IO-22U 通道 | oBIX 点名 | 用途 |
|---|------|---------|------------|-----------|------|
| 1 | 蜂鸣器 | 继电器 ON/OFF | DO-1 | `buzzer` | 烟雾告警 |
| 2 | 警示 LED | 继电器 ON/OFF | DO-2 | `warning_led` | 噪声告警 |
| 3 | 照明 LED | 0-10V 调光 | AO-1 | `lighting_led` | 自适应照明 |
| 4 | 风扇 | 继电器 ON/OFF | DO-3 | `fan` | 模拟空调 |

## 二、系统需要实现的功能

### 功能 1：实时数据采集与存储

- 每 2 秒从 oBIX 读取全部 8 路传感器数据
- 根据电流变送器读数自动计算风扇实时功率
- 所有数据存入本地数据库，保留最近 7 天
- 具备数据校验：读数超出传感器量程时丢弃并记录异常

### 功能 2：安全告警（最高优先级）

> 安全规则必须在 JACE8000 本地和 PC 端双重执行，确保 PC 宕机时仍能独立告警。

**烟雾 → 蜂鸣器**：
- 烟雾浓度 > 阈值（默认 150ppm）→ 蜂鸣器立即 ON，记录告警
- 烟雾浓度 < 阈值 − 回滞（默认 130ppm）→ 蜂鸣器 OFF，解除告警

**噪声 → 警示 LED**：
- 噪声 > 阈值（默认 55dB）持续 3 秒 → 警示 LED ON，记录告警
- 噪声 < 阈值 − 回滞（默认 52dB）持续 10 秒 → 警示 LED OFF，解除告警

> 安全告警自动执行，不经过 AI 决策流程。只有告警解除后，设备才恢复自动模式。

### 功能 3：空间占用感知（FSM 状态机）

根据多传感器数据实时判断图书馆空间是否有人，输出四种状态：

| 状态 | 含义 | 设备策略 |
|------|------|---------|
| VACANT（空闲） | 无人 | 照明/风扇全关 |
| ARRIVING（到达） | 检测到有人进入 | 触发 AI 初始化评估 |
| OCCUPIED（占用） | 人员稳定存在 | AI 持续优化环境 |
| LEAVING（离开） | 人员正在离开 | 执行关闭序列 |

**判断依据**——多传感器加权评分（权重总和 100%）：
- CO₂ 浓度：35%（最可靠的人员存在信号）
- 噪声水平：20%
- 温度变化率：15%
- 湿度变化率：10%
- PM2.5：10%
- 光照变化率：10%

> 烟雾传感器不参与占用判断——它是安全信号，不影响舒适性决策。但烟雾超标时系统强制进入告警状态。

### 功能 4：AI 智能决策（双引擎）

**引擎一 — FSM 状态机**：零成本本地毫秒级响应，判断"有没有人"。

**引擎二 — LLM 大模型决策**（仅在 ARRIVING/OCCUPIED 状态激活）：
- 综合室内传感器 + 室外天气 + 用户偏好模式
- 输出照明亮度建议（0-100%）+ 风扇启停建议 + 中文决策理由
- LLM 不可用时自动降级为 FSM 本地规则，保证系统持续运行

### 功能 5：用户偏好模式切换

系统支持三种运行模式，一键切换：

| 模式 | 开风扇温度 | 开灯照度 | 照明亮度 | 适用场景 |
|------|-----------|---------|---------|---------|
| 节能优先 | >28°C | <200 Lux | 50% | 寒暑假、深夜 |
| 均衡模式（默认） | >26°C | <300 Lux | 60% | 日常运营 |
| 舒适优先 | >24°C | <500 Lux | 80% | 考试季、重要活动 |

### 功能 6：手动控制覆盖

支持通过 Dashboard 手动控制任意设备：
- 蜂鸣器/警示 LED/风扇：强制 开 / 关 / 恢复自动
- 照明 LED：设置亮度 0-100% / 恢复自动
- 手动模式不参与自动控制，直到用户切换回自动模式

### 功能 7：能耗统计

- 实时显示风扇功率（W）和累计耗电（kWh）
- AI 模式 vs 常开模式能耗对比
- 折算碳减排量（kg CO₂）
- 支持按小时/天/周查看

### 功能 8：AI 对话助手

参考往届优秀作品，增加独立的 AI 对话助手功能：

- **对话页面**：用户可输入文字与 AI 自由对话
- **流式响应**：AI 回答实时逐字显示，带进度指示
- **可中断**：支持中途停止 AI 回答
- **快捷提问按钮**：
  - "分析当前环境数据"——AI 总结传感器状态并给出建议
  - "查看近期趋势"——AI 分析历史数据变化
  - "查询应急预案"——AI 根据当前告警给出处置步骤
- **上下文感知**：AI 知道当前传感器数据、FSM 状态、活跃告警

### 功能 9：语音输入

- 浏览器端语音识别（中文），将语音转为文字输入
- 主要用于 AI 对话助手的语音提问
- 有录音状态指示（波形动画 / 指示灯）
- 语音识别失败时有明确提示

### 功能 10：告警弹窗

烟雾告警触发时，不仅蜂鸣器响，前端也弹出醒目的告警弹窗：

- 显示：告警类型、当前传感器数值、触发阈值
- 附带简要处置指引（如"请检查烟雾来源，确认是否为火情"）
- 告警解除后弹窗自动消失
- 噪声告警可设为横幅提示即可，不阻断操作

### 功能 11：天气显示

主控看板展示当前室外天气信息：

- 温度、天气状况（晴/阴/雨等）、湿度、风速
- 数据来源：OpenWeather 等免费天气 API
- 与室内传感器数据同屏对比展示

### 功能 12：Demo 测试模式

硬件未连接时，系统可切换到 Demo 模式开发和演示：

- 使用预录的传感器数据模拟真实数据流
- 数据随时间有合理波动（模拟人员进出、光照变化等）
- 一键切换真实模式 / Demo 模式
- 方便队友在没有硬件时并行开发前后端

### 功能 13：数据导出

- 历史趋势页支持导出为 CSV 文件
- 图表支持导出为 PNG 图片
- 能耗数据支持导出

### 功能 14：应急预案知识库

AI 助手内置应急处置知识，涵盖常见异常场景：

- 火情/烟雾超标 → 蜂鸣器已触发，同时 AI 可给出疏散建议
- 噪声持续超标 → 提醒管理员介入
- CO₂ 浓度过高 → 建议开窗或加强通风
- PM2.5 异常升高 → 建议检查空调滤网或室外污染源
- 系统断连 → 提示检查 JACE8000 和 IO-22U 连接

> 注：AI 只提供文字建议，不自动执行安全相关动作。安全动作由功能 2 的安全规则独立执行。

### 功能 15：键盘快捷键（竞赛演示用）

竞赛演示时可以通过键盘快速触发场景：

- 按键强制切换 FSM 状态（模拟人员进出）
- 按键快速触发/解除告警（演示告警响应）
- 按键切换 Demo 模式 / 真实模式

## 三、后端需要提供的 API

> 以下列出 API 的功能，具体路径和参数格式由后端同学自行设计。

| # | API 功能 | 说明 |
|---|---------|------|
| 1 | 获取所有传感器实时数据 | 返回 8 路传感器 + 风扇功率的当前值、单位、时间戳 |
| 2 | 获取传感器历史数据 | 按传感器名 + 时间范围查询，返回时序数据供图表使用 |
| 3 | 获取设备状态 | 返回 4 个执行器的当前开关/亮度/模式状态 |
| 4 | 手动控制设备 | 下发控制指令到指定设备 |
| 5 | 获取系统状态 | 返回 FSM 当前状态、AI 模式、活跃偏好模式、LLM 推理文字、运行时长 |
| 6 | 切换偏好模式 | 在节能/均衡/舒适三种模式间切换 |
| 7 | 获取能耗汇总 | 返回耗电量、风扇运行时长、碳减排量、AI vs 常开对比 |
| 8 | 获取活跃告警 | 返回当前未解除的告警列表（类型、严重级别、触发值） |
| 9 | AI 对话 | 发送用户消息，返回 AI 流式响应（SSE），支持中断 |
| 10 | 获取天气数据 | 返回当前室外天气（温度、状况、湿度、风速） |
| 11 | 获取应急预案 | 根据告警类型返回对应处置建议 |
| 12 | 切换 Demo 模式 | 在真实数据和模拟数据之间切换 |
| 13 | 导出数据 | 按传感器和时间范围导出 CSV 数据 |

## 四、前端需要的页面

### 页面 1：主控看板

这是竞赛演示的核心页面，需要展示：

- **顶部**：系统标题、实时时钟、AI 模式指示灯（绿=正常/黄=降级/红=告警）、当前室外天气
- **传感器区**：8 张传感器卡片 + 1 张风扇功率卡片，数值实时刷新，颜色随健康度变化
- **状态区**：FSM 四种状态的可视化指示灯，当前状态高亮
- **设备区**：4 个执行器状态面板，含开关状态和手动/自动模式标识
- **AI 区**：LLM 最近一次决策的推理文字
- **告警横幅 + 弹窗**：有活跃告警时红色滚动横幅，烟雾告警时弹出详情弹窗
- **Toast 通知**：操作反馈以小弹窗形式出现，自动消失

### 页面 2：AI 助手

独立的对话式 AI 助手页面：

- **对话区**：聊天气泡形式展示对话历史，AI 回复支持 Markdown 渲染
- **输入区**：文字输入框 + 语音输入按钮
- **语音状态**：录音中显示波形动画或指示灯
- **快捷按钮**：分析环境、查看趋势、查询应急预案
- **流式显示**：AI 回复逐字出现，可中途点击停止
- **上下文**：每次提问自动附带当前传感器数据和系统状态

### 页面 3：历史趋势

- **传感器选择**：勾选需要查看的传感器（支持多选叠加）
- **时间范围**：预设 1 小时 / 6 小时 / 24 小时 / 7 天，也支持自定义
- **图表**：多传感器折线图叠加，不同传感器可共用或独立 Y 轴
- **导出**：支持将当前图表数据导出为 CSV 文件

### 页面 4：能耗分析

- **概览卡片**：今日耗电(kWh)、风扇运行时长(h)、碳减排量(kg)
- **对比图**：AI 模式 vs 常开模式能耗柱状图
- **功率曲线**：24 小时风扇功率面积图
- **占比图**：照明/风扇能耗环形图
- **模式对比**：三种偏好模式的预估年化能耗对比表

## 五、数据存储要求

需要持久化存储以下数据：

| 数据类别 | 存储内容 | 保留时长 |
|---------|---------|---------|
| 传感器时序数据 | 传感器名 + 数值 + 单位 + 时间戳 | 7 天 |
| 设备操作日志 | 设备名 + 操作 + 来源（AI/手动/安全规则）+ 时间 | 7 天 |
| FSM 状态记录 | 状态名 + 评分 + 时间 | 7 天 |
| LLM 决策记录 | 状态 + 推理文字 + 决策动作 + 时间 | 永久 |
| 告警记录 | 类型 + 级别 + 消息 + 触发/解除时间 | 永久 |

## 六、整体架构

```
传感器 (4-20mA)
    │
    ▼
IO-22U (Modbus RTU)
    │
    ▼
JACE8000 (oBIX 服务)
    │  HTTP/XML
    ▼
PC 后端 (Python)
    ├── oBIX 数据采集线程（2s 周期）
    ├── FSM 状态机线程（2s 周期）
    ├── 安全规则引擎（每轮必执行）
    ├── LLM 决策引擎（按需激活）
    ├── LLM 对话服务（SSE 流式）
    ├── 天气服务（OpenWeather）
    ├── Demo 数据生成器
    ├── SQLite 数据库
    └── Flask API 服务
        │  RESTful JSON
        ▼
    浏览器 Dashboard
    ├── 主控看板（传感器 + 设备 + 告警 + 天气）
    ├── AI 助手（对话 + 语音输入 + 应急预案）
    ├── 历史趋势（多传感器图表 + 导出）
    └── 能耗分析（对比图 + 碳减排）
```

## 七、关键参数

以下参数供后端实现时参考，可根据实际传感器标定调整：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 烟雾告警阈值 | 150 ppm | 超过触发蜂鸣器 |
| 烟雾回滞 | 20 ppm | 防反复触发 |
| 噪声告警阈值 | 55 dB | 超过触发警示 LED |
| 噪声回滞 | 3 dB | 防反复触发 |
| 噪声防抖时间 | 3 秒 | 超过阈值需持续此时间才触发 |
| 噪声恢复确认 | 10 秒 | 低于阈值需持续此时间才解除 |
| 传感器轮询周期 | 2 秒 | |
| FSM 评估周期 | 2 秒 | |
| 数据保留天数 | 7 天 | |
| 风扇电压 | 24V DC | 功率 = 24V × 电流 |
